网上支付接口文档V2.01

1.接口介绍

1.1.产品介绍

  系统提供商户通过系统对接的方式发起移动支付交易，包括微信，支付宝。

  商户通过系统对接后，可使用到在线支付产品(包含网关、微信、支付宝、银联等支付方式)，可以让消费者在网上商城安全、便捷地购买产品或享受服务，且商家可以更及时地了解款项的到帐情况。


  商户接入流程：

  ◆ 商户提交入网资料

  ◆ 平台审核通过

  ◆ 平台开通相关产品业务

  ◆ 商户登录商户综合管理系统查看商户号以及密钥

  ◆ 进行接口联调接入

1.2.支付模式

1）网关支付

  目前支持个人网银支付：个人用户通过自己所拥有的借记卡、信用卡的银行，在申请开通网上银行之后，进行网上支付的过程，称为个人网银在线支付。

2）扫码支付

  扫码支付是指商户按指定流程生成支付二维码后，用户再用“扫一扫”功能完成支付的模式。

3）刷卡支付

  刷卡支付是用户展示钱包内的“刷卡条码/二维码”给商户系统扫描后直接完成支付的模式。主要应用线下面对面收银的场景。

4）H5支付

  H5支付是用户在商户的H5页面中选择支付方式，然后商户H5页面调起支付APP进行支付，用户最终完成支付。

2.接口规则

2.1.协议规则

  ◆ 传输方式：HTTP传输

  ◆ 提交方式：采用POST方式提交

  ◆ 字符编码：统一采用UTF-8字符编码方式

  ◆ 数据格式：返回数据为JSON格式

  ◆ 签名算法：MD5

  ◆ 签名要求：请求和异步数据均需进行校验签名，详细签名方法请见 2.2.签名机制

2.2.签名机制

  本接口中，签名参数定义为：sign，以下为使用MD5加密，生成sign字符串的方法：

1）除sign字段外，所有非空参数值的参数按照参数名ASCII码从小到大排序（字典序），使用URL键值对的格式拼接成待签名字符串stringA

mch_id=84000000000
total_fee=100
service=HYR_ZFB_WEB
out_trade_no=2018073112221
sign_type=MD5
notify_url=http://www.baidu.com
body=test
return_url=http://www.baidu.com

stringA="body=test&mch_id=84000000000&notify_url=http://www.baidu.com&out_trade_no=2018073112221&return_url=http://www.baidu.com&service=HYR_ZFB_WEB&sign_type=MD5&total_fee=100";

以下规则请着重注意：

  ◆ 参数名ASCII码从小到大排序（字典序）；

  ◆ 如果参数的值为空不参与签名；

  ◆ 参数名区分大小写；

  ◆ 接口可能增加字段，验证签名时必须支持增加的扩展字段。

2）在stringA后面拼接上key得到stringB字符串，并对stringB进行MD5运算，得到sign值

  示例：sign=md5(stringB.getbyte("utf-8")).toUpperCase()

stringB=stringA+"&key=ab68ed046100bb80ec8af66231d657b150222";

sign=md5(stringB.getByte("utf-8")) .toUpperCase()="C16E9891DC3CF79211A6DECA5FEB56B7";

  （注：key获取路径：商户综合管理系统→订单管理→用户信息）

3.接口定义

3.1.支付接口

3.1.1.业务功能

  买家在商户网站拍下商品后，选择相应付款方式，商户系统向支付平台发送请求数据，平台根据不同支付方式支付信息，用户进行支付。用户支付成功后，服务器后台异步通知商户服务器（notify_url）支付结果。

3.1.2.接口地址

  URL地址：http://202.239.38.240:8081/superpay/api/v1/transRequest

3.1.3.请求参数

参数名称 参数含义 取值 是否必填 最大长度 备注
mch_id 商户号 平台分配的商户号 是 15 登录商户系统可查
total_fee 交易金额 单位:分 是 12 金额是1元则填 100
service 交易类型 交易类型 是 12 附录4.1交易类型
out_trade_no 交易单号 商户的交易订单号 是 32 每笔交易唯一
sign_type 签名方式 MD5 是 3 MD5
notify_url 回调地址 商户接收回调地址 是 60
body 商品名称 订单商品名称 是 80
return_url 跳转地址 支付成功后跳转地址 是 60
bankno 银行类型 交易银行的简称 否 10 直连网银必填，参见附录4.2银行简码
cardNo 交易卡号 银行卡 否 30 直连网银、快捷支付必填
sign 签名 参见2.2签名机制 是 32

3.1.4.响应参数

参数名称 参数含义 取值 是否必填 最大长度 备注
ret_code 返回码 SUCCESS/FAIL 否 8 通信标识，非交易结果
ret_message 返回码说明 否 100
以下信息只有当ret_code为SUCCESS时有返回
payinfo 支付链接 如果是扫码支付则返回二维码字符串 是 不限
orderNo 交易单号 平台订单号 否 32
total_fee 交易金额 交易金额 否 12 单位:分

3.2.订单查询接口

3.2.1.业务功能

  根据商户交易单号查询该商户订单的支付结果。

3.2.2.接口地址

  URL地址：http://202.239.38.240:8081/superpay/api/v1/tradeQuery

3.2.3.请求参数

参数名称 参数含义 取值 是否必填 最大长度 备注
mch_id 商户号 平台分配的商户号 是 15
out_trade_no 订单号 平台订单号 是 32
sign 签名 参见2.2签名机制 否 32

3.2.4.同步响应参数

参数名称 参数含义 取值 是否必填 最大长度 备注
ret_code 返回码 SUCCESS/FAIL 否 8 通信标识，非交易结果
ret_message 返回码说明 否 100
以下信息只有当ret_code为SUCCESS时有返回
orderNo 交易单号 平台的交易流水号 否 20
payStatus 交易状态 支付的状态 否 4 附录4.4.支付状态码
total_fee 交易金额 交易金额 否 12 单位:分

3.3.异步通知参数

参数名称 参数含义 取值
mch_id 商户号 平台分配的商户号
out_trade_no 商户流水号 系统内唯一
service 交易方式
tradeStatus 支付状态 附录4.4.支付状态码
total_fee 交易金额 单位：分
sign 签名信息 参见2.2签名机制

通知响应结果：

  商户服务端在接收到异步通知并正确处理后，需要给支付平台响应“success”（注：大小写敏感，请使用小写 success ，不要添加其他任何字符串），其他响应或者不响应通联开发平台则通过重发机制进行重发，重发次数为3次。

  注意:后期可能会增加通知参数,请获取全量变量,然后遍历获取非空变量,而不能用request.getparam(“key”)这样用文档中的字段名来一个个获取,避免增加通知字段后,无法兼容。

代码示例：

private TreeMap getParams(HttpServletRequest request) {
    TreeMap map = new TreeMap();
    Map reqMap = request.getParameterMap();
    for (Object key : reqMap.keySet()) {
        String value = ((String[]) reqMap.get(key))[0];
        log.info("接受参数:{}={}", key, value);
        map.put(key.toString(), value);
    }
    return map;
}
